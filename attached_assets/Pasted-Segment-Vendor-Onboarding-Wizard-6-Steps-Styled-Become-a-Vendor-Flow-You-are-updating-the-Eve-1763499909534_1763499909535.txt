Segment: Vendor Onboarding Wizard (6 Steps, Styled) + Become a Vendor Flow

You are updating the EventHub app built with React + TypeScript + TailwindCSS.

GOAL
1. Implement a 6-step vendor onboarding wizard with this sidebar order:
   1) Service Type
   2) About You
   3) Location
   4) Portfolio
   5) Service Description
   6) Ready to Create Listing?

2. Use this same wizard when:
   - A user signs up directly as a vendor, OR
   - A logged-in customer clicks “Become a Vendor” in the profile dropdown.

3. After completing Step 6, save all data, upgrade the user to vendor (if needed), and redirect them to their Vendor Dashboard.

---------------------------------------------------
GLOBAL STYLE

Apply these styles consistently across all onboarding steps:

- White background
- Primary color: #9EDBC0
- Cards and main containers:
  - rounded-xl
  - light shadow
- Use a centered main card for each step with:
  - Header (title)
  - Short description (optional)
  - Form fields
  - Navigation buttons (“Back”, “Next”, etc.)

Left sidebar:
- Title: “Vendor Profile”
- Vertical list of the 6 steps with numbered circles.
- Active step highlighted (e.g., primary color circle and darker text).
- Completed steps can show a subtle checkmark or filled circle.

---------------------------------------------------
ENTRY POINTS

A) New Vendor Signup
- When a user signs up as a vendor, navigate them to: `/vendor/onboarding`.
- Start at **Step 1: Service Type**.

B) “Become a Vendor” (Existing Customer)
- In the profile dropdown for logged-in customers, there is an item: **“Become a Vendor”**.
- When clicked:
  - Navigate to `/vendor/onboarding`.
  - Start at **Step 1: Service Type**.
- On successful completion of the wizard:
  - Upgrade their role to `vendor`.
  - Ensure they have a vendor dashboard record.
  - Redirect to `/vendor/dashboard`.

Both paths must share the same onboarding wizard and logic.

---------------------------------------------------
STEP COMPONENTS (MAPPING TO FILES)

Use/organize the steps as separate components. Map them like this:

- Step 1 → `Step1_ServiceSetup.tsx`  (Service Type)
- Step 2 → `Step2_AboutYou.tsx`     (create this if it doesn’t exist yet)
- Step 3 → `Step3_Location.tsx`     (Location)
- Step 4 → `Step4_Portfolio.tsx`    (rename from Photos component if needed)
- Step 5 → `Step5_ServiceDescription.tsx`
- Step 6 → `Step6_ReadyToCreateListing.tsx`

Each step component should receive props for:
- current form state
- setters/update handlers
- navigation handlers (onNext, onBack)

Persist data in a parent component (`VendorOnboarding.tsx`) that controls the active step and passes state down.
