CHUNK 2 – Vendor Model + Onboarding
Title: Vendor Model + Onboarding
Prompt:
Extend the event marketplace to support vendor onboarding.
	1. Create vendors table linked to users (1 vendor per vendor user).
	2. Fields: business_name, description, service_area, categories, base_price, deposit_amount, media (array or separate table), status (draft | submitted | approved), calendar_url.
	3. Add availability table with fields: vendor_id, date, is_available.
	4. Build a Vendor Dashboard (protected for vendor role):
		○ Edit and save profile info
		○ Upload images (store file paths)
		○ Manage availability (block/unblock dates)
		○ Show listing status (draft/submitted/approved).
	5. Include an endpoint for vendors to “submit for approval” (changes status).
	6. Document how to create, edit, and view a vendor profile.

CHUNK 3 – Customer Browsing + Event Planner
Title: Customer Browsing + Event Planner

Create public browsing and guided event-planning flow.

		○ Step 2: select vendor types (Venue, Photographer, Florist, DJ, Catering, Prop Rentals)
		○ Step 3: follow-up questions for at least Florist and Venue (color palette, flowers, budget, indoor/outdoor, guest count)

CHUNK 4 – Booking + Stripe Deposits
Title: Booking + Stripe Deposits
Prompt:
Implement booking creation and Stripe integration for deposits.
	1. Create bookings table with:
		○ id, customer_id, vendor_id, event_date, total_price, deposit_paid, status (pending_payment | confirmed | cancelled), stripe_payment_id, remaining_balance_due_date.
	2. Booking flow:
		○ Customer selects date/time on vendor page.
		○ Backend checks vendor availability.
		○ Create booking record with status pending_payment.
		○ Redirect to Stripe Checkout for the vendor’s deposit_amount.
	3. On Stripe success webhook:
		○ Mark booking as confirmed, deposit_paid = true.
		○ Set remaining_balance_due_date = event_date - 14 days.
	4. Log successful payment in console for MVP (email notifications optional).
	5. Document how to test with Stripe’s test keys and sample checkout.
